Author

Date
2014 This class encapsulates the OSC message-based communication between a Streaming Audio Manager (SAM) and a third-party renderer.

int sam::SamRenderer::addType ( int  id,
const char *  name,
int  numPresets,
int *  presetIds,
const char **  presetNames 
)

Add a rendering type and its presets.

This method informs SAM that the type and preset are available for use. The "basic" type (id 0) with "default" preset (id 0) are always available and therefore do not need to be added explcitly.

Parameters
idthe unique ID for the rendering type being added
namethe user-readable name for the type
numPresetsthe number of presets this type has (including the required default)
presetIdsan array of length numPresets containing the unique ID for each preset (including the required default)
presetNamesan array of length numPresets containing the user-readable name for each preset (including the required default)

void sam::SamRenderer::responseReceived ( )
signal

Emitted when a response from SAM is received.

This signal is for use internal to the SamRenderer library, and users don't need to (and shouldn't) connect it to any slots.

int sam::SamRenderer::unsubscribeToPosition ( int  id)

Unsubscribe to changes in position information for the stream with given ID.

The renderer is automatically unsubscribed when a stream is removed, so this is only needed if the renderer wishes to unsubscribe while a stream is still active.
